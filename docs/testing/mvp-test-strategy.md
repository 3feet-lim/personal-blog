# MVP 테스트 전략

## 범위

이 전략은 MVP에서 위험도가 가장 높은 경로를 보호하기 위한 것입니다.

- 공개 블로그 탐색
- 가족 앨범 인가
- 보호 자산 정책
- 관리자 전용 관리 접근
- PostgreSQL 호환성을 고려한 SQLite 우선 로컬 동작

## 고위험 경계

### 1. 인증과 인가의 분리

위험:

- 사용자가 인증에는 성공했지만 가족 콘텐츠를 보면 안 되는 상황이 발생할 수 있습니다.

검증 범위:

- 인증은 되었지만 승인되지 않은 사용자가 가족 라우트에서 거부되는지 확인합니다.
- 가족 권한이 승인된 사용자가 가족 라우트에 접근 가능한지 확인합니다.
- 관리자가 모든 보호 경로와 관리자 경로에 접근 가능한지 확인합니다.

### 2. 보호 자산 접근

위험:

- 비공개 가족 자산이 공개 URL로 접근 가능해질 수 있습니다.

검증 범위:

- 비공개 자산이 가족 권한 또는 관리자 권한 없이는 접근되지 않는지 확인합니다.
- 공개 블로그 자산은 가족 권한 없이도 접근 가능한지 확인합니다.

### 3. SQLite와 PostgreSQL 이식성

위험:

- ORM과 스키마 선택이 SQLite에서는 통과하지만 PostgreSQL 전제를 깨뜨릴 수 있습니다.

검증 범위:

- 초기 스키마는 이식 가능하게 유지하고, MVP 핵심 경로에서 데이터베이스 전용 기능을 피합니다.
- 영속성 복잡도를 늘리기 전에 PostgreSQL 상에서 호환성 검사를 수행합니다.

## 권장 자동화 범위

### 단위 테스트

- 인가 정책 헬퍼
- 저장소 공개/비공개 규칙
- 설정 파싱과 환경 변수 기본값

### 통합 테스트

- `GET /api/blog/posts`
- `GET /api/blog/posts/{slug}`
- `GET /api/albums`
  - 익명 요청
  - 인증됨, 미승인 요청
  - 가족 승인 요청
  - 관리자 요청
- `GET /api/admin/users`
  - 일반 사용자 요청
  - 관리자 요청

### 엔드투엔드 스모크 테스트

- 익명 사용자가 홈과 블로그 페이지를 탐색할 수 있어야 합니다.
- 승인된 가족 사용자가 앨범 페이지에 접근할 수 있어야 합니다.
- 승인되지 않은 사용자는 앨범 페이지에서 거부되어야 합니다.
- 관리자는 관리자 페이지에 접근할 수 있어야 합니다.

## 인수 기준

- 공개 블로그 콘텐츠는 로그인 없이 보여야 합니다.
- 가족 앨범 콘텐츠는 승인 없이 차단되어야 합니다.
- 관리자 전용 API는 비관리자 사용자를 거부해야 합니다.
- 보호 자산 메타데이터는 공개/비공개 규칙을 지켜야 합니다.

## 추후 검증 범위

- 동영상 미디어 흐름
- 앨범별 세부 인가
